you're usually NOT the only one

Often we think, "I'm probably the only one that ran into this problem" we are usually wrong. In programming for instance there's thousands of people out there trying to do the same things, the more programmers there are the more likely your crazy 1 in a million state is one they'll run into as well.

This is why documentation is so important, because writing lasts forever, so long as it's not ruined.
